## 3. 인증 체계

```
사용자 요청
    │
    ▼
middleware.ts ── /admin/* 경로 가드
    │            (admin_session 쿠키 확인)
    │            미인증 → /admin/login 리다이렉트
    │
    ▼
/admin/login ── 비밀번호 입력
    │           POST /api/admin/auth
    │           → authenticateUser(password)
    │           → 쿠키 세팅 (base64 인코딩된 세션)
    │           → sessionStorage에 role/name 저장
    │
    ▼
API 라우트 ── verifyAdmin(req)
              1순위: 쿠키(admin_session) → decodeSession → role=admin 확인
              2순위: Bearer Token → ADMIN_PASSWORD 매칭 (하위호환)
```

**역할 기반 접근**:
- `admin` — 전체 접근
- `branch` — 지점 관리자 (제한적 접근)

**환경변수**:
- `ADMIN_PASSWORD` — 단일 비밀번호 (레거시)
- `ADMIN_USERS` — JSON 배열 (다중 사용자: `[{name, password, role, branch}]`)

### 5-1. 마케팅 이메일 시스템

```
┌─ 유입 경로 ─────────────────────────────────────────────────┐
│                                                              │
│  DemoModal 제출 ──→ /api/lead ──→ 자동 구독자 등록 [NOTE-24] │
│  Contact 문의  ──→ /api/lead ──→ 자동 구독자 등록            │
│  Footer 구독   ──→ /api/newsletter/subscribe → 직접 등록     │
│  관리자 수동   ──→ /api/admin/subscribers → 직접 등록         │
│                                                              │
└──────────────────────────────────────────────────────────────┘
         │
         ▼
┌─ 구독자 DB (data/subscribers.json) ─────────────────────────┐
│  id, name, email, org, role, size, phone                     │
│  tags[]         ← 성향 세그먼트 (PRESET_TAGS 18종)          │
│  status         ← active / unsubscribed                      │
│  optInAt        ← 수신 동의 일시 (법적 증빙)                │
│  unsubscribedAt ← 수신 거부 일시                             │
│  source         ← demo_modal / contact_page / newsletter     │
└──────────────────────────────────────────────────────────────┘
         │
         ▼
┌─ 이메일 발송 (/api/admin/email/send) ───────────────────────┐
│  1. 태그 기반 대상 필터링 (OR 조건, 빈=전체)                 │
│  2. {name}, {org}, {role} 변수 치환 (개인화)                 │
│  3. EMAIL_WEBHOOK_URL → 외부 서비스 발송                     │
│     (미설정 시 시뮬레이션 모드: 콘솔 로그만)                 │
│  4. 캠페인 이력 저장 (data/email-campaigns.json)             │
└──────────────────────────────────────────────────────────────┘
         │
         ▼
┌─ 수신거부 (/api/newsletter/unsubscribe) ────────────────────┐
│  GET ?email=xxx → 원클릭 수신거부 + HTML 확인 페이지         │
│  POST {email}   → JSON 응답                                  │
│  ※ 데이터 삭제 아님, status 변경 (법적 보존)                │
└──────────────────────────────────────────────────────────────┘
```

**관리자 UI** (`/admin/marketing`):
- **구독자 관리 탭**: 목록·태그 뱃지·상태 필터·수동 추가·삭제
- **이메일 발송 탭**: 제목/본문 작성, 태그 대상 선택, `{name}` 미리보기, 발송
- **발송 이력 탭**: 캠페인별 상태·수신자 수·대상 태그
