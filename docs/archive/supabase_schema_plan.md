## 📝 1. 스키마(엑셀 헤더) 설계 내역

### 1-1. `blog_posts` (블로그 게시글) 테이블
어드민 패널의 기존 UI와 환상적으로 호환되도록 구성하였으며, `category`는 확장성과 작업 속도를 고려하여 프론트엔드 통제(단순 문자열) 방식을 채택했습니다.

| 항목(Column) | 타입(Type) | 비고 |
| :--- | :--- | :--- |
| **`id`** | **UUID** | 무작위 고유 식별자 (PK) |
| **`title`** | text | 글 제목 |
| **`excerpt`** | text | 리스트 등에서 보여질 요약/미리보기 내용 |
| **`category`** | text | "인사이트", "제품 업데이트" 등 (프론트 단에서 배열로 통제) |
| **`tag`** | text | 게시물 관련된 해시태그 |
| **`author`** | text | 작성자명 |
| **`authorRole`** | text | 마케터, 원장 등 직책 |
| **`readTime`** | text | "3 min read" |
| **`imageUrl`** | text | 썸네일 이미지 주소 |
| **`featured`** | boolean | 메인 노출 여부 (기본값: false) |
| **`published`** | boolean | 작성 완료 및 공개 여부 (기본값: false) |
| **`date`** | timestampz | 최초 생성/작성 날짜 |
| **`deletedAt`** | timestampz | 휴지통(Soft Delete) 보관용 날짜. 없다면 정상 노출됨 |
